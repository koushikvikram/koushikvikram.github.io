Feature Selection for Machine Learning

Feature Selection is the process of selecting a subset of relevant features(variables, predictors) for use in Machine Learning model building.

Feature Selection = Search Technique for proposing new feature subsets + Evaluation Measure.

Feature Selection is computationally expensive. Also, different feature subsets render optimal performance for different machine learning algorithms. Due to these reasons, many different methods for feature selection have been developed to accommodate as many caveats and limitations as possible.

Feature selection is important because
  • Simpler models are easier to interpret (a model with 10 variables is easier for the user to interpret than a model with 100 variables)
  • Simpler models in turn have shorter training and inference times, which is important in a live environment.
  • Enhanced generalization by reducing overfitting (removing noisy variables and irrelevant features enhances generalization)
  • Easier to implement by software developers. Less Variables -> Less code -> Less Bugs
  • Reduced risk of data errors during model use.
  • Variable Redundancy can be avoided. Redundant variables do not provide any new information to the model.
  • High dimensional spaces causes poor performance of models, especially tree-based models. Reducing the feature space helps build more robust models.
Feature Selection can be divided into three main categories:
  • Filter Methods
    • Variance
    • Correlation
    • Univariate selection
  • Wrapper Methods
    • Forward Selection
    • Backward Selection
    • Exhaustive Search
  • Embedded Methods
    • LASSO
    • Tree Importance
